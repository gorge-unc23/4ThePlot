# 4ThePlot Functional Requirements

| FR_01 | 4ThePlot shall provide landing, login, stored-session, and role-based navigation for Goer, Business, and Admin users. | After authentication, the bottom navigation and first screen must match the signed-in user's role. | 1 | 26/05/2026 | Angjelina Zenunaj / Briana Llapaj |
| FR_02 | The system shall allow visitors to register as Goer or Business users with role-specific signup fields and image selection. | Goers provide interests, while Business users provide business profile details and logo information. | 1 | 26/05/2026 | Ilion Elezaj / Klaus Saliaj |
| FR_03 | The system shall provide profile, settings, appearance preference, announcements, and logout flows. | Users access these actions from the Profile and Settings screens, with non-admin users able to view platform announcements. | 1 | 26/05/2026 | Gëzime Mirku / Ronald Lamani |

| FR_05 | The system shall display event details with event image, title, status, date and time, location, host, capacity, categories, tags, comments, share, report, and join controls. | The event details page is the main visible decision point before joining, reporting, sharing, or opening the host profile. | 1 | 26/05/2026 | Briana Llapaj / Ilion Elezaj |
| FR_06 | The system shall allow eligible Goer users to join and unregister from events with visible capacity handling. | Join and unregister controls must update the event registration state shown on the event details page. | 1 | 26/05/2026 | Klaus Saliaj / Gëzime Mirku |
| FR_07 | The system shall provide calendar views for joined Goer events and hosted Business events, including city filtering for hosted events. | Calendar dates must show event indicators and selected-date event lists based on the current user's role. | 2 | 26/05/2026 | Ronald Lamani / Revi Beja |
| FR_08 | The system shall provide a map view for visible event locations. | Goer users see available event locations, while Business users can focus on hosted event locations. | 2 | 26/05/2026 | Angjelina Zenunaj / Ilion Elezaj |
| FR_09 | The system shall support event comments, including loading comments, posting comments, reporting comments, and admin-visible deletion where supported. | Comment actions appear on the event details page and depend on the user's role and relationship to the comment. | 1 | 26/05/2026 | Briana Llapaj / Klaus Saliaj |
| FR_10 | The system shall allow users to submit safety reports for events, comments, and hosts from visible report controls. | Report dialogs collect a reason and severity before submitting reports for admin review. | 1 | 26/05/2026 | Gëzime Mirku / Revi Beja |
| FR_11 | The system shall allow users to submit, view, and delete host verification requests from Settings. | Verification requests support PDF document upload and display request status in the Verification settings page. | 2 | 26/05/2026 | Ronald Lamani / Briana Llapaj |
| FR_12 | The system shall allow Business users to manage hosted events through hosted list, create event, upload event image, edit event, and delete event flows. | Business event management is available through the Hosted tab, Add flow, event details host actions, and edit flow. | 1 | 26/05/2026 | Angjelina Zenunaj / Briana Llapaj |
| FR_13 | The system shall display business or host profile information and a hosted event list from host/profile views. | Users can open a host profile from event details and inspect other events hosted by that profile. | 2 | 26/05/2026 | Ilion Elezaj / Klaus Saliaj |
| FR_14 | The system shall allow Business users to promote a hosted event by marking it as trending. | The visible Promote event flow updates the event so it can appear in the Discover trending section. | 2 | 26/05/2026 | Gëzime Mirku / Ronald Lamani |
| FR_15 | The system shall provide an Admin dashboard and Admin-specific tab navigation. | Admin users see dashboard tiles for moderation, verification, notifications, metrics, disputes, audit logs, events, map, and profile. | 1 | 26/05/2026 | Revi Beja / Angjelina Zenunaj |
| FR_16 | The system shall allow Admin users to view and moderate all events, including deleting events. | Admin event moderation is accessible from the dashboard and Events tab. | 1 | 26/05/2026 | Briana Llapaj / Ilion Elezaj |
| FR_17 | The system shall allow Admin users to review safety reports, filter reports, apply moderation actions, and update report status. | Report detail screens show reporter, target, reason, evidence, and action controls with reason dialogs. | 1 | 26/05/2026 | Klaus Saliaj / Gëzime Mirku |
| FR_18 | The system shall allow Admin users to review host verification requests and approve or reject verification decisions. | Host verification detail screens show request status, host information, documents, and decision controls. | 1 | 26/05/2026 | Ronald Lamani / Revi Beja |
| FR_19 | The system shall allow Admin users to create and edit global notifications and allow non-admin users to view published announcements. | Admin notification forms collect title, message, status, schedule, and reason; user settings show visible announcements. | 2 | 26/05/2026 | Angjelina Zenunaj / Ilion Elezaj |
| FR_20 | The system shall allow Admin users to monitor metrics, review disputes, view dispute chat logs, resolve disputes, and view audit logs. | Admin operational screens support growth KPIs, dispute evidence review, dispute decisions, and recorded platform actions. | 1 | 26/05/2026 | Briana Llapaj / Klaus Saliaj |
