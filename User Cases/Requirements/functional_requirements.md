# 4ThePlot Functional Requirements

| Req# | Requirement | Comments | Priority | Date | Reviewed/Approved |
| :--- | :--- | :--- | :---: | :---: | :--- |
| FR_01 | 4ThePlot shall provide role-based login and separate views for Goer, Business, and Admin users. | Different dashboards and permissions are shown after authentication based on role. | 1 | 03/04/2026 | Angjelina Zenunaj / Briana Llapaj |
| FR_02 | The system shall allow Goer and Business users to create events and save them as drafts. | Draft events remain hidden from public discovery until the host publishes them. | 1 | 03/04/2026 | Ilion Elezaj / Klaus Saliaj |
| FR_03 | The system shall allow event hosts to set a maximum attendee capacity per event. | The capacity value controls the maximum number of confirmed joins. | 1 | 03/04/2026 | Gëzime Mirku / Ronald Lamani |
| FR_04 | The system shall prevent extra joins when an event is full and shall offer waitlist enrollment. | Users are added to a queue and can be notified when seats become available. | 1 | 03/04/2026 | Revi Beja / Angjelina Zenunaj |
| FR_05 | The system shall support recurring event setup with weekly or monthly repetition. | Hosts configure frequency and end conditions once instead of duplicating entries manually. | 2 | 03/04/2026 | Briana Llapaj / Ilion Elezaj |
| FR_06 | The system shall allow event hosts to send broadcast updates to all joined attendees. | A single message is sent to all participants for schedule changes or instructions. | 2 | 03/04/2026 | Klaus Saliaj / Gëzime Mirku |
| FR_07 | The system shall personalize discovery results using followed categories for each goer. | Recommended events should match saved interests such as music, sports, or art. | 2 | 03/04/2026 | Ronald Lamani / Revi Beja |
| FR_08 | The system shall display host ratings, reviews, and verification status on event details. | Trust indicators are visible before users decide to join paid or free events. | 2 | 03/04/2026 | Angjelina Zenunaj / Ilion Elezaj |
| FR_09 | The system shall support opening event addresses in map navigation and sharing event links. | Users can get directions quickly and invite contacts through share actions. | 3 | 03/04/2026 | Briana Llapaj / Klaus Saliaj |
| FR_10 | The system shall provide admin tools for report moderation, verification, announcements, disputes, and growth monitoring. | Admin actions cover platform safety, trust, communication, and operational monitoring. | 1 | 03/04/2026 | Gëzime Mirku / Revi Beja |
